## 一、准备工作：认识 Terminal（终端）

### 1.1 「终端」是什么？

终端（Terminal）就是你跟电脑**用文字对话的窗口**。平时你用鼠标点点点，现在你只需要**敲几个单词**，电脑就能听懂并执行。是不是很酷？

### 1.2 怎么打开它？

在 Mac 上，最简单的方法是：

> 🔍 按 `Command + 空格键`（Spotlight 搜索），输入 **`terminal`**，然后按回车。

duang~ 一个黑底白字（也可能是白底黑字）的窗口就弹出来了。别被它吓到，它只是在等你说话。

> 💡 **小贴士**：如果你觉得默认的字体太小，可以按 `Command + +` 放大，`Command + -` 缩小。

#### 🪟 Windows 用户怎么打开终端？

在 Windows 上，有几种方式：

| 方法 | 操作 | 推荐度 |
|------|------|--------|
| **搜索打开** | 按 `Win 键`，输入 **`powershell`** 或 **` terminal`**，回车 | ⭐⭐⭐ 最推荐 |
| **运行对话框** | 按 `Win + R`，输入 **`cmd`**，回车 | ⭐⭐ 传统方式 |
| **右键开始菜单** | 右键点击开始按钮 → 选择「Windows PowerShell」或「终端」 | ⭐⭐ |

> 🪟 **建议 Windows 用户使用 PowerShell 或 Windows Terminal**，不要用旧的 CMD（命令提示符）。因为 PowerShell 支持更多现代命令，和本指南里讲的 Mac 命令几乎一样。

> 🪟 **另一个好选择：Git Bash**。安装 Git for Windows 时会附带一个 Git Bash 终端，它完全支持本指南里的所有 Mac/Linux 风格命令（`ls`、`cd`、`mkdir` 等），无需额外学习。

### 1.3 你必须会的 6 条命令

下面这些命令就像你在"文件管理器"里拖来拖去一样，只不过换成了文字。每条命令都遵循这个模式：

```
命令 [选项] [目标]
```

来，一条一条认识它们：

---

#### 📂 `ls` — 看看这里有什么

```bash
ls
```

**作用**：列出当前文件夹里的所有文件和子文件夹。

| 常用变体 | 含义 |
|----------|------|
| `ls` | 列出当前目录的所有文件 |
| `ls -l` | 显示详细信息（大小、日期） |
| `ls -a` | 包括隐藏文件（以 `.` 开头的） |
| `ls -la` | 详细信息 + 隐藏文件，最常用！ |

---

#### 🚪 `cd` — 走进一个文件夹

```bash
cd 文件夹名
```

**作用**：切换当前所在的文件夹（就像双击进入一个文件夹）。

| 常用写法 | 含义 |
|----------|------|
| `cd Desktop` | 进入桌面 |
| `cd ..` | 返回上一层 |
| `cd ~` | 回到你的"家"目录（home） |
| `cd /` | 去电脑的最顶层（根目录） |

> 🧠 **理解路径**：
> - `~` = `/Users/你的用户名`，也就是你的"家"
> - `.` = 当前文件夹
> - `..` = 上一层文件夹
> - 就像你要去朋友家，"家 → 出小区 → 坐地铁 → 到站"一样，路径就是你的"地址"。

---

#### 📁 `mkdir` — 新建一个文件夹

```bash
mkdir 新文件夹的名字
```

```bash
mkdir 作业文件夹
```

执行后，当前目录就会出现一个叫「作业文件夹」的新文件夹。

---

#### 📝 `cp` — 复制文件

```bash
cp 源文件 目标位置
```

```bash
cp 报告.txt ~/Desktop/
```

↑ 把「报告.txt」复制到桌面。

| 常用变体 | 含义 |
|----------|------|
| `cp -r 文件夹 目标` | 复制整个文件夹（`-r` = 递归） |

---

#### 🚚 `mv` — 移动 / 重命名

```bash
mv 源文件 目标位置
```

**双重身份**：
- 如果目标是另一个目录 → 移动
- 如果目标是一个新名字 → 重命名

```bash
mv 报告.txt ~/Documents/     # 移动到「文稿」文件夹
mv 旧名字.txt 新名字.txt     # 重命名
```

---

#### 🗑️ `rm` — 删除文件

```bash
rm 文件名
```

> ⚠️ **注意！** 终端里删掉的文件不经过废纸篓，删了就真没了！下手前先确认！！

```bash
rm 没用的草稿.txt            # 删一个文件
rm -r 整个文件夹/             # 删一个文件夹（慎重！）
rm -rf 整个文件夹/            # 强制删除（更慎重！！）
```

---

#### 🪟 Windows 用户：命令对照表

如果你用的是 **PowerShell** 或 **Windows Terminal**，上面讲的 Mac 命令**全部可以直接用**！`ls`、`cd`、`mkdir`、`cp`、`mv`、`rm` 在 PowerShell 里都能正常执行。

如果你用的是老式的 **CMD（命令提示符）**，对照关系如下：

| 功能 | Mac / PowerShell | CMD（旧版） | 说明 |
|------|-----------------|------------|------|
| 查看文件 | `ls` | `dir` | PowerShell 两种都支持 |
| 切换目录 | `cd` | `cd` | 完全一样 ✅ |
| 新建文件夹 | `mkdir` | `mkdir` 或 `md` | 完全一样 ✅ |
| 复制文件 | `cp` | `copy` | PowerShell 两种都支持 |
| 移动/重命名 | `mv` | `move` | PowerShell 两种都支持 |
| 删除文件 | `rm` | `del` | PowerShell 两种都支持 |

> 🪟 **强烈建议用 PowerShell 或 Git Bash**，这样你学一套命令就能在 Mac 和 Windows 上通用了。本指南后续所有命令示例都基于 Mac/PowerShell 风格。

### ✅ 小测试

打开你的 Terminal，试试这个练习：

```bash
cd ~/Desktop              # 去桌面
mkdir claude-test         # 建一个叫 claude-test 的文件夹
cd claude-test            # 进去
ls                        # 看看里面有什么（应该是空的）

```

### 1.4 理解 Mac 上的文件路径（中文系统 ≠ 中文路径）

你的 Mac 操作系统可能是中文的，Finder 里看到的文件夹名字也是中文的——「桌面」「下载」「文稿」「图片」……但！在 Terminal 里，<strong>它们叫的是英文名字</strong>：

| Finder 里看到的中文名 | Terminal 里的英文路径 | 完整路径 |
|----------------------|---------------------|---------|
| 桌面 | `Desktop` | `~/Desktop` 或 `/Users/你的用户名/Desktop` |
| 下载 | `Downloads` | `~/Downloads` |
| 文稿 | `Documents` | `~/Documents` |
| 图片 | `Pictures` | `~/Pictures` |
| 音乐 | `Music` | `~/Music` |
| 影片 | `Movies` | `~/Movies` |
| 应用程序 | `Applications` | `/Applications` |

> 🧠 **记住这个规则**：你在 Finder 里看到的是「替身」（显示名），Terminal 认的才是「真名」（路径名）。<strong>路径永远用英文</strong>。

#### 🪟 Windows 上的路径

Windows 的路径格式和 Mac 不太一样，但也不难理解：

| 概念 | Mac | Windows |
|------|-----|---------|
| 根目录 | `/` | `C:\`（或者 D:\、E:\ ...） |
| 家目录 | `~` 即 `/Users/你的用户名` | `C:\Users\你的用户名` |
| 桌面 | `~/Desktop` | `C:\Users\你的用户名\Desktop` |
| 下载 | `~/Downloads` | `C:\Users\你的用户名\Downloads` |
| 分隔符 | `/`（斜杠） | `\`（反斜杠）**或** `/`（斜杠也行） |

> 🪟 **好消息**：在 PowerShell 和 Git Bash 里，你可以继续用 `/` 作为分隔符，不需要纠结 `\`！比如 `cd ~/Desktop` 在 Windows PowerShell 里一样能跑。Claude Code 内部也统一用 `/` 处理路径。

#### 路径里的 `/` 是什么意思？

`/` 就像文件夹之间的「分隔符」，和网址里的 `/` 是一个道理：

```
~/Downloads/test
```

翻译成人话：**「家 → 下载 → test（文件夹或文件）」**

- `test` 可能是**文件夹**，也可能是**文件**——光看路径没法区分，用 `ls` 看一眼就知道了
- 如果有后缀（比如 `test.txt`、`test.py`、`test.html`），那八九不离十是个文件

#### 路径命名小技巧

给文件和文件夹起名字的时候，尽量：
- ❌ 不要用中文（有些工具会翻车）
- ❌ 不要带空格（`我的 作业.txt` → Terminal 里要写成 `我的\ 作业.txt` 或者加引号）
- ✅ 用英文 + 下划线/连字符（比如 `homework_1.txt`、`course-project`）

### 1.5 好习惯：一个项目，一个文件夹

在开始用 Claude Code 做某个课程作业之前，强烈建议你养成这个习惯：

```bash
# 第 1 步：给每个课程/项目建一个专属文件夹
mkdir ~/Desktop/数字信号处理作业

# 第 2 步：cd 进去
cd ~/Desktop/数字信号处理作业

# 第 3 步：在这个文件夹里启动 Claude
claude
```

**为什么要这样做？** 🎯

- Claude 的「视野」以你启动它时所在的文件夹为中心——它更容易看到这个文件夹里的文件
- 不会跟其他课程的文件搞混
- 聊天的上下文（历史记录）跟项目绑定，下次回来继续聊更清楚
- 以后找文件也方便，不会满桌面都是乱七八糟的文件

> 💡 **进阶小贴士**：如果你想把某个文件夹里已有的文件交给 Claude 看，用第三节讲的 `@` 符号就行。

## 二、启动 Claude Code

### 2.1 确认已安装 Claude Code

在 Terminal 里输入：

```bash
claude --version
```

如果显示版本号（比如 `v1.0.x`），说明已经装好了。跳去 2.3。

如果提示 `command not found: claude`，跟着 2.2 来装。

### 2.2 安装 Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

> 如果提示 `npm: command not found`，说明你的 Mac 还没装 Node.js。别急，最简单的办法是：
>
> 1. 打开 [nodejs.org](https://nodejs.org) 
> 2. 下载左边那个 **LTS** 版本（就是推荐给大多数用户的那个）
> 3. 双击 `.pkg` 文件安装，一路"继续"
> 4. 装好后**关掉 Terminal 重新打开**
> 5. 再试 `npm install -g @anthropic-ai/claude-code`

### 2.3 启动！

```bash
claude
```

回车之后，**第一次使用**会弹出浏览器让你登录 Anthropic 账号。照提示操作就行（可以用 Google 账号登录）。

登录成功后回到 Terminal，你会看到类似这样的界面：

```
┌─────────────────────────────────────────────┐
│  Claude Code v1.x.x                         │
│  Type /help for available commands          │
│                                             │
│  >                                           │
└─────────────────────────────────────────────┘
```

那个 `>` 就是在等你说话的地方，你可以直接输入中文跟 Claude 聊天！比如：

```
帮我写一份课程作业的提纲
```

---

> 💡 **小贴士**：如果哪天你想退出 Claude Code，输入 `/exit` 或者按 `Ctrl + C` 几次就行。

### 2.4 如何退出 Claude Code

有几种方式可以结束对话：

| 方式 | 操作 | 效果 |
|------|------|------|
| 礼貌退场 | 输入 <code>/exit</code> | 优雅退出，对话历史保留 |
| 强退 | 按 <code>Ctrl + C</code> **两次** | 相当于"我不聊了，走人"，对话历史也会保留 |
| 一次 Ctrl+C | 按一下 | 只是打断当前正在进行的操作，不会退出 |

> 💡 **建议**：习惯用 **双击 `Ctrl + C`** 退出，简单快捷。退出后你回到 Terminal 的黑窗口，可以接着输入别的命令。

### 2.5 重新进入之前的对话（网页端历史对话）

你某天聊了一半退出了，第二天想继续聊？用这个：

```bash
# 先 cd 到你的项目文件夹
cd ~/Desktop/数字信号处理作业

# 然后启动 Claude
claude

# 在 Claude Code 的输入框里输入：
/resume
```

<strong>`/resume`</strong> 会列出一个历史对话列表，你可以用上下箭头选择之前的对话，按回车继续聊！

就像微信的聊天记录一样——上次聊到哪，这次接着聊，Claude 还记得你们之前说过什么。

> 💡 **注意**：`/resume` 显示的是你在这个**文件夹**里启动过的对话。所以 1.5 小节说的「一个项目一个文件夹」的好习惯在这里又用上了——不同课程项目的对话不会混在一起。

### 2.6 常用的 Claude Code 内置命令

在 Claude Code 的输入框里，以 `/` 开头的都是内置命令。下面是最常用的几个：

| 命令 | 作用 | 什么时候用 |
|------|------|----------|
| `/help` | 查看所有可用命令 | 忘了命令的时候 |
| `/exit` | 退出 Claude Code | 聊完了，走人 |
| `/resume` | 查看并恢复历史对话 | 继续上次没聊完的 |
| `/clear` | 清空当前对话上下文 | 话题跑偏了，想重新来过 |
| `/cost` | 查看本轮对话消耗的 token | 想知道花了多少额度 |
| `/status` | 查看当前会话状态 | 看看 Claude 在干嘛 |
| `/add-dir` | 添加文件夹到 Claude 的视野 | 让它看到更多文件 |

> 💡 你可以随时输入 `/help` 获取最新的命令列表。

### 2.7 遇到任何问题？直接告诉 Claude！

这是最重要的一条原则——<strong>你不需要真的会写代码，你只需要会"告诉 Claude 发生了什么"</strong>。

| 你遇到的情况 | 你直接对 Claude 说 | Claude 会 |
|-------------|-------------------|-----------|
| 运行报错了 | "这段代码报错了：`[粘贴错误]`，请帮我修" | 诊断错误并修复 |
| 想装某个工具 | "我想用 ffmpeg 处理视频，但我没装，请帮我装" | 搜索并安装 |
| 不知道怎么做某件事 | "我想把 CSV 数据画成图表，我完全不会，请一步步教我" | 写代码 + 解释 |
| 代码跑得太慢 | "这个程序跑得太慢了，能帮我优化吗？" | 分析并优化 |
| 心情不好 😤 | "这破代码怎么都调不通，我要崩溃了" | 安慰你 + 帮你调 |

> 🎯 **记住**：Claude Code 不仅仅是写代码的工具，它就像坐在你旁边的**全能助教**——能帮你装东西、改错误、解释概念、甚至听你吐槽。你只管把问题描述清楚，<strong>不需要你真的会写代码</strong>。

### 2.8 写错了 / 按错了？别慌——ESC 是你的「后悔药」

在用 Claude Code 的时候，你可能会遇到这些尴尬时刻：

| 场景 | 你的内心 OS |
|------|-----------|
| 打了一长串话，发现写得不好 | "完了，能不能撤回……" |
| 还没打完，不小心按了 Enter 发出去了 | "等下我还没说完！！" |
| Claude 开始回答了，但你突然想改需求 | "不对不对，我不是这个意思！" |
| 觉得这次的回答不满意，想回到之前重来 | "这答得不行，我想倒回去……" |

**解决方法超级简单——按一下 `ESC` 键！**

| 操作 | 效果 |
|------|------|
| 按 **1 次 `ESC`** | 🛑 中断当前对话 / 打断 Claude 正在说的话 |
| 按 **多次 `ESC`** | ⏪ 弹出「回退到哪一步」的选择框，可以一步步倒退到之前的对话节点 |

> 🎮 你可以把 `ESC` 理解成**游戏里的「暂停 + 读档」按钮**：
> - 按一下 → 暂停当前回合
> - 多按几下 → 弹出存档列表，选一个回去

这样你永远不用害怕「说错话」——反正能撤回！

### 2.9 Claude Code 实用快捷键速查

#### Claude Code 专属快捷键

| 快捷键 | 作用 | 使用场景 |
|--------|------|---------|
| `ESC` | 中断 / 回退对话 | 写错了、不满意、想回到之前 |
| `Alt + Enter` | 换行（不发送） | 写多行内容时，按 Enter 不会发送，另起一行继续写 |
| `Alt + V` | 粘贴图片 | 把截图粘贴到对话里让 Claude 看（比描述一万句都强！） |

> 📸 **关于粘贴图片**：如果代码报错了，直接截个图 `Alt+V` 贴进去，比手动复制错误信息快多了。Claude 能「看」懂图片里的内容。

#### Terminal 通用快捷键（在 Claude Code 里也能用）

> ⚠️ **Mac 和 Windows 快捷键不同！** 下面列出的是 **Mac** 上的快捷键。如果你用的是 Windows，`Cmd` 换成 `Ctrl` 就行（除了 `Ctrl+U/A/E/W` 这类控制键，它们在两个平台都一样）。

| 快捷键 (Mac) | 快捷键 (Windows) | 作用 | 使用场景 |
|-------------|-----------------|------|---------|
| `Cmd + C` | `Ctrl + C` | 复制选中文本 | — |
| `Cmd + V` | `Ctrl + V` | 粘贴文本 | — |
| `Ctrl + U` | 同左 | 快速删除当前行 | 打的字全不要了，一秒清空 |
| `Ctrl + A` | 同左 | 跳到行首 | 想在开头加东西 |
| `Ctrl + E` | 同左 | 跳到行尾 | 想在末尾继续写 |
| `Ctrl + W` | 同左 | 删除前一个单词 | 快速删掉刚才打错的一个词 |
| `Ctrl + K` | 同左 | 删除光标到行尾 | 删掉后半行 |
| `Ctrl + Y` | 同左 | 粘贴刚才删除的内容 | "咦，删多了，还给我！" |

> 💡 **小贴士**：`Ctrl + U` 简直是救命快捷键——当你打了一大堆指令发现全错了，不用狂按退格键，直接 `Ctrl + U` 一秒清空！

> 🍎 **Mac 用户特别注意**：在 Mac Terminal 里，`Ctrl + C` 是**中断进程**（不是复制！）——这也是为什么 2.4 小节说用 `Ctrl + C` 两次退出 Claude。复制粘贴请用 `Cmd + C` / `Cmd + V`。

## 三、让 AI 读你的文件

Claude 很聪明，但它不会"自动看到"你电脑上的文件。你需要**手动告诉它**要读哪些。

### 3.1 操作步骤（超级简单）

1. 在 Claude Code 的输入框里，先**敲一个空格**
2. 然后输入 **`@`**（就是 shift+2 那个符号）
3. 屏幕下方会弹出一个文件列表，用**上下箭头键**选择文件
4. 按**回车**选中

就像这样：

```
> 请帮我检查这段代码有什么问题 
```

（输入空格 + @ 后，弹出文件选择器）

```
> 请帮我检查这段代码有什么问题 @作业1.cpp
```

### 3.2 可以选多个文件

在输入 `@` 之后，你可以重复这个操作，选择多个文件一起给 Claude 看。比如：

```
> 请帮我分析 @src/main.py 和 @src/utils.py 之间的关系
```

### 3.3 为什么要先让 Claude 读文件？

举个栗子 🌰：你不能对一个没看过你论文的人说"帮我改一下第三章"。同样的道理，Claude 需要**先看到你的文件内容**，才能给你有用的建议。

> 💡 **还记得吗？** 如果你忘了要读的文件在哪个文件夹，先用 `ls` 和 `cd` 在 Terminal 里逛一圈，找到目标文件，记下路径和名字，再去 Claude Code 里引用它。

## 四、让 Claude 帮你做事的核心技巧

### 4.1 魔法咒语 🔮

你的电脑可能没装什么特别的软件——这很正常！Claude 可以帮你**自动搜索、安装、调用**需要的工具。你只需要在提示词里加上类似的话：

> **"请搜索/调用/安装相关的 skills 或 GitHub 仓库来完成这个任务"**

### 4.2 举个例子

❌ **不够好的说法**：
```
帮我画一个数据图
```

✅ **更好的说法**：
```
帮我用 Python 的 matplotlib 画一个数据图。如果没有安装 matplotlib，请先帮我安装。
如果缺少合适的 skills，请搜索并调用相关的 MCP 工具或 GitHub 仓库来完成。
```

### 4.3 Claude 能帮你做的事

| 你需要的 | 你可以说 |
|----------|----------|
| 安装 Python 包 | "请先安装 xxx，再写代码" |
| 下载 GitHub 项目 | "请从 GitHub 上 clone xxx 仓库" |
| 调用特定工具 | "请安装并使用 pandoc 把文件转成 PDF" |
| 搜索已有轮子 | "请搜索有没有现成的库可以处理这个任务" |

### 4.4 让 Claude 安装 GitHub 仓库

有时候你想要的工具不是 `pip install` 就能装的——它可能是一个 GitHub 上的开源项目。别担心，Claude 照样能帮你搞定！

#### 案例：安装 claude-hud（一个 Claude Code 的增强界面插件）

```
请帮我从 https://github.com/jarrodwatts/claude-hud 安装这个工具。
请帮我完成以下步骤：
1. clone 这个仓库到本地
2. 按照它的 README 说明安装依赖
3. 配置好让它能在 Claude Code 中使用
```

Claude 会自己去看那个仓库的 README，搞清楚安装方法，然后一步步帮你弄好——你不需要自己去读文档。

> 🎯 **通用模板**：
> ```
> 请帮我安装这个 GitHub 仓库：[粘贴仓库地址]
> 如果需要额外安装依赖，请一并帮我装好。
> ```

### 4.5 让 Claude 安装 Skills

Skills（技能）是 Claude Code 的「插件」，安装之后 Claude 就多了对应的能力。

#### 案例：安装 Anthropic 官方 Skills 集合

```
请帮我从 https://github.com/anthropics/skills 安装这些 skills。
请按照仓库的说明，把所有可用的 skills 配置到我的 Claude Code 中。
```

安装完成后，Claude 就能使用这些 skill 来处理更复杂的任务了。

> 💡 **什么是 Skills？** 可以把它们想象成给 Claude 安装的「专业培训」——没装之前 Claude 什么都会一点，装了对应的 skill 之后它在某个领域就更专业了。

> 💡 **核心思路**：不要假设 Claude "可能做不到"。你只管把需求说清楚，工具的事让它自己想办法！

## 五、关于 .md 文件（Markdown）——用啥软件打开？

### 5.1 什么是 Markdown？

Markdown（简称 `.md`）是一种**超级简单的排版语言**。你不用像 Word 那样用鼠标点"加粗"、"标题"按钮，只需要在文字前后加几个符号就行。

比如你看到的本文档，就是 Markdown 写的！对比一下：

| 你想要的效果 | Markdown 写法 | 最终显示 |
|-------------|-------------|---------|
| 大标题 | `# 这是大标题` | 又大又粗的标题 |
| 二级标题 | `## 这是二级标题` | 稍小的标题 |
| 加粗 | `**加粗的文字**` | **加粗的文字** |
| 斜体 | `*斜体*` | *斜体* |
| 列表 | `- 项目一` | • 项目一 |
| 链接 | `[文字](网址)` | 可点击的超链接 |
| 代码 | `` `代码片段` `` | `代码片段` |
| 代码块 | ` ``` 代码 ``` ` | 带高亮的代码框 |

> 🎯 **重点**：Claude Code 输出的内容很多时候就是 Markdown 格式！所以你需要知道怎么查看它。

### 5.2 用什么软件打开 .md 文件？

你的 Mac 上有好几种方式可以查看 Markdown 文件，从简单到专业：

---

#### 🌟 方案一（推荐）：Visual Studio Code（VS Code）

这是程序员最爱用的编辑器，**免费又好用**。

1. 去 [code.visualstudio.com](https://code.visualstudio.com) 下载安装
2. 用 VS Code 打开 `.md` 文件
3. 按 `Command + Shift + V` → **右边会弹出实时预览窗口**，左边是源码，右边是排版效果

> ✅ **优点**：免费、功能强大、有侧边栏预览、还能顺便编辑代码

---

#### 📝 方案二（最简单）：Mac 自带的「快速查看」

**不需要装任何软件！**

1. 在 Finder（访达）里找到 `.md` 文件
2. **选中文件** → 按一下**空格键**
3. 弹出一个预览窗口，就能看到排版后的效果了

> ✅ **优点**：零安装、速度最快
> ⚠️ **缺点**：只能看不能编辑，有些复杂格式显示不太好

---

#### 🖊️ 方案三：Mac 自带的「文本编辑」

1. 双击 `.md` 文件，用「文本编辑」打开
2. 你会看到 Markdown 的**源代码**（带 `#` `*` 等符号的那种）
3. 可以编辑文字

> ⚠️ 「文本编辑」不会渲染 Markdown 效果，你看到的是"源码"。但改文字没问题。

---

#### 🎨 方案四（最漂亮）：Typora

如果你想让 Markdown 看起来就像 Word 一样美观：

1. 去 [typora.io](https://typora.io) 下载
2. 用 Typora 打开 `.md` 文件
3. **所见即所得**——你打字的时候它直接显示效果，不用切换预览

> ✅ **优点**：界面极简、所见即所得、导出 PDF/HTML 方便
> ⚠️ **缺点**：正式版需要付费（约 $15），但可以一直用试用版

---

### 5.3 快速对比

| 软件 | 费用 | 预览方式 | 适合谁 |
|------|------|---------|--------|
| **快速查看**（空格键） | 免费 | 按空格预览 | 只想瞄一眼 |
| **文本编辑** | 免费 | 无预览 | 简单改几个字 |
| **VS Code** | 免费 | `Cmd+Shift+V` 侧边预览 | 🔥 **最推荐** |
| **Typora** | 付费 | 所见即所得 | 经常写文档 |

### 5.4 Claude 输出 Markdown 时你该怎么做

1. Claude 输出 Markdown 内容后，**把内容全部复制**
2. 用 VS Code（或其他编辑器）新建一个文件
3. 粘贴，保存为 `xxx.md`
4. 按 `Command + Shift + V`（VS Code）或按空格键（Finder）查看效果

> 💡 **小贴士**：如果你想让 Claude 把 Markdown 直接转成 HTML，可以说：**"请把上面的 Markdown 内容输出为单个 HTML 文件，内嵌所有 CSS 和 JS"**，然后双击打开就能在浏览器里看到漂亮排版了！

## 六、输出 LaTeX 文档（课程作业必备）

> 很多理工科课程要求交排版精美的作业，LaTeX 就是为此而生的。别怕，你不需要学 LaTeX 语法——Claude 帮你写，你只需要编译。

### 5.1 什么是 `.tex` 文件？

`.tex` 是 LaTeX 的源文件，里面包含了你的**内容和排版指令**。把它"编译"一下就变成漂亮的 PDF。

### 5.2 让 Claude 写 LaTeX

告诉 Claude：

```
请帮我生成一份 LaTeX 文档（.tex 文件），内容是关于 xxx 的课程作业。
要求：支持中文（使用 ctex），有标题、作者、目录、章节，数学公式正确排版。
```

Claude 会输出一个完整的 `.tex` 文件代码片段。

### 5.3 如何编译 → 使用 texpage.com

因为你的电脑可能没装 LaTeX 编译器（那个东西好几 GB），推荐用在线网站 **[texpage.com](https://texpage.com)**：

#### 编译步骤：

```
第 1 步：打开浏览器，访问 https://texpage.com
第 2 步：点击右上角的「新建项目」(New Project)
第 3 步：给项目起个名字，比如「课程作业」
第 4 步：把 Claude 生成的 .tex 代码全部复制粘贴进去
第 5 步：点击顶部的「编译」(Compile / ▶ 按钮)
第 6 步：编译成功后，点击「下载 PDF」
```

> 📸 整个过程就像把文字复制到 Word 然后点"导出 PDF"一样简单！

### 5.4 完整的提示词模板

```
请帮我生成一份完整的 LaTeX 文档（.tex 文件），要求如下：
- 文档类型：article（带标题、作者、摘要、章节）
- 中文支持：使用 \usepackage[UTF8]{ctex}
- 内容：关于 [你的作业主题]
- 包含至少一个数学公式
- 包含一个表格
- 代码格式整洁，有适当的注释

请在代码开头的注释中写明：
% 将此文件内容复制到 https://texpage.com 即可在线编译
```

### ✅ 小测试

用上面的模板让 Claude 生成一份 LaTeX，然后去 texpage.com 编译，看看能不能成功导出 PDF！

> 💡 **小贴士**：如果编译报错，**把错误信息复制给 Claude**，它能帮你修。就像这样：`@编译报错.txt 这个 LaTeX 编译时报了这些错误，请帮我修复`。

## 七、输出交互式 HTML 演示文档

课程展示、答辩汇报、数据报告……很多时候你需要一个**好看又能在浏览器打开**的东西。

### 7.1 魔法咒语 🎩

告诉 Claude 这句话，效果立竿见影：

> **"请输出单个 HTML 文件，内嵌所有 CSS 和 JS 代码，无任何外部依赖"**

| **无外部依赖** | 不需要联网加载任何东西，断网也能用 |

### 7.3 完整模板

```
请帮我生成一个交互式演示网页，要求：
- 输出单个 HTML 文件，内嵌所有 CSS 和 JS 代码，无任何外部依赖

### 7.4 如何使用

Claude 会给你一长串代码 → 你把它**全部复制** → 粘贴到一个文本编辑器（Mac 自带的「文本编辑」就行）→ **保存为 `.html` 文件**（比如 `演示.html`）→ 双击打开！

## 九、为什么 Claude 老问我问题？——Plan Mode 说明

### 9.1 第一次见到它，你可能懵了

用着用着，你可能会看到上方显示 **`"1 plan mode"`**，然后界面弹出选项让你选。比如：

```
┌─────────────────────────────────────────────────┐
│  Claude wants to:                                │
│  创建 3 个文件，修改 2 个文件                      │
│                                                  │
│  [Accept]  [Reject]  [Modify]                    │
└─────────────────────────────────────────────────┘
```

你心里可能在想：**"我就让你写个作业，怎么还让我做选择？？"**

别急，这是好事。

### 9.2 Plan Mode 是什么？

Plan Mode 是 Claude 的 **「先想好再动手」** 模式。数字（比如 N = 1、2、3…）表示这是第几个计划。

它就像一个**严谨的施工队**——在砸墙之前，先给你看图纸：

> "老板，我打算在这面墙上开个窗，图纸是这样的。你看行不行？行的话我就动手了。"

### 9.3 三个选项的含义

| 按钮 | 含义 | 什么时候选 |
|------|------|----------|
| **Accept**（同意） | "行，就按你说的做吧" | 方案看起来合理 |
| **Reject**（拒绝） | "不行，方向不对" | 完全不是你想要的 |
| **Modify**（修改方案） | "方向差不多，但需要改改" | 大体 ok，细节要调 |

### 9.4 为什么这是好事？

🎯 **保护你的文件**：Claude 不会乱来——它先告诉你它打算做什么，你确认了它才动手。不会出现"我让它改第3章，它把整篇论文重写了"的惨剧。

🎯 **帮你把关**：有时候你可能随口说了个模糊的需求，Plan Mode 让 Claude 先"翻译"一遍给你看，你可以及时纠正。

### 9.5 如果你不想每次都选...

如果你信任 Claude，可以直接在输入框里加上：

> **"Please proceed"** 或 **"请直接执行"**

Claude 就会跳过 Plan Mode，直接开干。

同理，如果你想**进入** Plan Mode：

> **"先给我一个计划，我确认后再执行"**

> 💡 **建议**：遇到**重要任务**（改论文、改代码、删文件）时，让 Claude 先在 Plan Mode 里列出方案给你看，确认无误再执行。遇到**简单任务**（问个问题、查个资料），直接说 "请直接执行" 省时间。
